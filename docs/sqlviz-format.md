# The `.sqlviz` File Format

The `.sqlviz` extension is the native workspace save format for **SQL Visualize**. It allows you to persist your drag-and-drop query schema layouts and restore them across different sessions.

## 🗄️ Format Structure

A `.sqlviz` file is purely a JSON structure generated by the React Flow engine under the hood. It stores spatial coordinates, node connections, and local configuration logic. 

**It contains two primary arrays:**

### 1. `nodes` 
An array of objects representing the tables on your canvas.
- **`id`**: Unique identifier (e.g., `dndnode_1`).
- **`position`**: Spatial X/Y coordinates on your infinite canvas.
- **`data`**: Internal business logic:
  - `tableName`: The full signature of the SQL server table.
  - `tableAlias`: Custom alias specified for the query generation.
  - `columns`: Array of columns containing `name`, `type`, `isSelected`, aggregation `function` details, `alias` fields, and the newly added `filter` objects (WHERE conditions and operators).

### 2. `edges`
An array of objects representing the data relationships (JOINs) between tables.
- **`source`** / **`target`**: Linked Node IDs.
- **`sourceHandle`** / **`targetHandle`**: Exact Column connections ensuring Type-Safe validations.
- **`data`**: Features the `joinType` (e.g., `INNER`, `LEFT`, `RIGHT`) configured via the visual Venn diagrams.

---

## 🔒 Security & Portability

- **No Passwords**: The `.sqlviz` file **NEVER** stores your database connection string, Server URI, Username, or Password. This guarantees it is safe to commit these files inside a target git repository or share them manually with team members. 
- **Schema Validation Engine**: If you load a `.sqlviz` file into a database where the schema has altered (e.g., table deleted, column renamed), SQL Visualize will read the file but execute a live comparison against `INFORMATION_SCHEMA`. Missing attributes will automatically be flagged with a `⚠️` visual warning, preventing unexpected SQL syntax execution failures.
